# PriceParser 🟡

Трекер цен на **Wildberries**, **Ozon** и **Яндекс.Маркете** с уведомлениями в Telegram при снижении цены.

> Учебный проект: веб-скрапинг, парсинг JS-сайтов, Selenium, работа с внутренними API маркетплейсов.

## Возможности

- 📉 Отслеживание цен по ссылке на товар (WB / Ozon / Яндекс.Маркет)
- 🎯 Уведомление при достижении **целевой цены** или **снижении на N%**
- 🗄️ История цен в SQLite
- 🤖 Уведомления в **Telegram** (или в консоль, если бот не настроен)
- ⏰ Периодическая проверка по расписанию (APScheduler)

## Архитектура

| Маркетплейс | Способ получения цены | Почему |
|-------------|----------------------|--------|
| **Wildberries** | Внутренний JSON-API `card.wb.ru/cards/v4/detail` | Быстро, без браузера, стабильно |
| **Ozon** | **Selenium** (`undetected-chromedriver`) | Агрессивный антибот + рендер цены через JS |
| **Яндекс.Маркет** | **Selenium** (`undetected-chromedriver`) | SmartCaptcha + рендер цены через JS |

```
priceparser/
  parser/    wb.py (API)  ·  ozon.py / yandex.py (Selenium)  ·  base.py (Parser, Product)
  storage/   db.py — SQLite: products + price_history
  notifier/  telegram.py — Telegram Bot API / ConsoleNotifier
  core/      tracker.py — логика проверки  ·  config.py — .env
  cli.py     add / list / remove / check / run
```

## Установка

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt          # для WB достаточно базовых пакетов
copy .env.example .env                    # заполнить токен Telegram (опционально)
```

> Для Ozon и Яндекс.Маркета дополнительно нужен Google Chrome — `undetected-chromedriver` скачает совместимый драйвер сам.

### Настройка Telegram (необязательно)

1. Создать бота у [@BotFather](https://t.me/BotFather) → получить `TELEGRAM_BOT_TOKEN`.
2. Узнать свой chat id у [@userinfobot](https://t.me/userinfobot) → `TELEGRAM_CHAT_ID`.
3. Вписать оба значения в `.env`. Без них уведомления печатаются в консоль.

## Графический экран

Всё в одном окне: таблица отслеживаемых товаров, поле добавления, кнопки и журнал проверок/уведомлений.

```powershell
python main.py ui
```

- Поле **«Ссылка / артикул»** + **«Цель ₽»** + выбор маркета (авто/wb/ozon/yandex) → **Добавить**
- **«Проверить цены»** — опрашивает все товары в фоновом потоке, результат и уведомления пишутся в **Журнал**
- Достигнутая цель подсвечивается зелёным; **«Удалить выбранное»** убирает товар
- **Превью**: выберите товар в таблице — справа загрузится его картинка (для WB строится из артикула, для Ozon/Яндекса берётся `og:image`)
- **Двойной клик** по строке открывает товар в браузере
- В поля можно **вставлять** (Ctrl+V) — работает и на русской раскладке; есть контекстное меню по правому клику (Вырезать/Копировать/Вставить)
- Сетевые операции идут в фоне — окно не подвисает

> Экран на Tkinter (входит в стандартную поставку Python — отдельных зависимостей не требует).

## CLI

```powershell
# добавить товар (маркетплейс определяется по ссылке) + целевая цена
python main.py add "https://www.wildberries.ru/catalog/68767845/detail.aspx" --target 2000

# добавить по артикулу
python main.py add 8056367 --marketplace wb

# Яндекс.Маркет (по ссылке маркетплейс определяется автоматически)
python main.py add "https://market.yandex.ru/product--slug/679402099"

python main.py list           # список отслеживаемого
python main.py remove 1       # убрать товар #1
python main.py check          # проверить цены один раз
python main.py run            # периодическая проверка (интервал из .env)
python main.py run --interval 600
```

Уведомление приходит, когда: цена ≤ целевой **или** цена снизилась относительно прошлой проверки минимум на `PRICE_DROP_THRESHOLD`.

## Настройки (`.env`)

| Переменная | Назначение | По умолчанию |
|------------|-----------|--------------|
| `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID` | доступ к боту | — (консоль) |
| `CHECK_INTERVAL` | интервал проверки, сек | `3600` |
| `PRICE_DROP_THRESHOLD` | минимальное снижение (0.05 = 5%) | `0.0` |
| `DB_PATH` | файл базы | `priceparser.db` |

## Статус

- ✅ WB-парсер (API), SQLite, Telegram, tracker, CLI, расписание — работает
- ✅ Ozon-парсер (Selenium) — реализован; селекторы Ozon периодически меняются, при сбое обновить `_PRICE_SELECTORS` в `parser/ozon.py`
- ✅ Яндекс.Маркет-парсер (Selenium) — реализован; при сбое обновить `_PRICE_SELECTORS` в `parser/yandex.py`
- ⚠️ **Антибот Ozon/Яндекса:** оба сайта детектят автоматизацию и/или блокируют по VPN/прокси (Яндекс прямо пишет «используете VPN»). Парсеры распознают страницу блокировки и сообщают причину (`BlockedError`). Что помогает: **выключить VPN/прокси**, запускать **не в headless** (`headless=False`), использовать residential-прокси. `undetected-chromedriver` 3.5.5 (стелс) **несовместим с Chrome 148** — закрывает окно; поэтому по умолчанию используется обычный Selenium. Включить uc принудительно: `set PRICEPARSER_USE_UC=1`.

## Ограничения и заметки

- Только для личного использования; соблюдайте rate-limit и ToS маркетплейсов.
- WB-API неофициальный — структура ответа может измениться (сейчас актуальна `v4`).
- Ozon может показывать капчу: при невозможности получить цену товар помечается как «нет в наличии».
